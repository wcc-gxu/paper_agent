# Paper Agent v3 — 产品规格说明

> 个人科研助理 Agent | 2026-06-14

---

## 1. 产品概述

Paper Agent v3 是一个**24/7 常驻后台的个人 AI 科研助理**。它不是一个"搜索论文的工具"，而是一个具备记忆、能主动发现知识、能自主决策的终身 Agent。

### 1.1 一句话描述

> 你的私人科研大脑——记住你研究过的一切，主动发现你该知道的东西。

### 1.2 产品形态

```
┌──────────────────────────────────────────┐
│              iOS 客户端                   │
│   对话 · 论文浏览 · 知识库 · 术语库 · 写作 │
├──────────────────────────────────────────┤
│    WebSocket (实时对话) + REST (数据查询)  │
├──────────────────────────────────────────┤
│       Agent 守护进程 (24/7 常驻)          │
│   主Agent + 6种子Agent (LangGraph)        │
│   意图理解 → Plan → 执行 → 验收 → 沉淀    │
├──────────────────────────────────────────┤
│  Semantic Scholar(P0) · PDF下载 · 转换    │
│  索引 · 聚类 · 去重 · 可视化 · 表达挖掘   │
├──────────────────────────────────────────┤
│  SQLite + ChromaDB + Redis(AOF) + 文件系统│
└──────────────────────────────────────────┘
```

## 3. 核心场景

### 场景 1：文献调研与筛选
**输入**：用户描述研究方向
**Agent 行为**：澄清需求 → 制定方案 → 多源搜索 → LLM 评估相关性 → 展示结果
**产出**：结构化论文列表 + 相关性评分 + 推荐理由

### 场景 2：文献综述生成
**输入**：确定的研究主题
**Agent 行为**：批量搜索 → 下载 PDF → 转换 MD → 向量索引 → 知识提取 → LLM 生成综述
**产出**：投稿级 Markdown 综述 + BibTeX 引用 + 知识卡片

### 场景 3：每日前沿追踪
**输入**：用户研究方向订阅（一次性设置）
**Agent 行为**：每天自动搜索 → 发现新论文 → 分析相关性 → 主动推送
**产出**：每日论文推送 + 摘要 + 推荐阅读理由

### 场景 4：论文精读与提炼
**输入**：用户选定的论文
**Agent 行为**：提取核心贡献 → 方法 → 数据集 → 局限 → 批判性分析
**产出**：结构化知识卡片 + 引用网络分析

### 场景 5：技术资料搜集（Phase 2）
**输入**：GitHub 仓库 / 官方文档 / Stack Overflow 等
**Agent 行为**：网页采集 → MD 转换 → 分类入库 → 语义检索
**产出**：结构化技术知识库

### 场景 6：入库进度查看
**输入**：用户确认调研计划后
**Agent 行为**：启动子Agent → 子Agent 编排入库全流程（搜索→评估→下载→转换→索引→排名→综述）→ 每步实时写入 JSON 日志
**iOS 展示**：入库进度页面实时显示当前阶段、每篇论文处理状态（下载中/已转换/已索引/失败原因）、预计剩余时间
**产出**：完整入库结果 + 结构化进度日志（task .jsonl 文件）

### 场景 7：智能去重（跨来源论文合并）
**输入**：多来源搜索结果（arXiv、Semantic Scholar、PubMed 等）或用户手动上传 PDF
**Agent 行为**：Embedding 向量相似度计算 → 发现 DOI 不同但内容相同的论文（预印本 vs 正式出版）→ 自动合并 → 标注关系
**产出**：去重后的论文库，每篇标注去重来源和替代版本
**iOS 展示**：去重报告（"合并了 5 组重复论文，节省 12 篇冗余"），重复组可展开查看详情

### 场景 8：研究方向自动聚类与聚焦
**输入**：项目中已入库的论文集合（如 50+ 篇）
**Agent 行为**：提取所有论文的 Embedding 向量 → K-means 聚类 → LLM 为每个聚类生成主题标签 → 生成研究方向地图
**产出**：研究主题分布图（t-SNE/UMAP 可视化）、每个主题的核心论文和关键词
**iOS 展示**：交互式研究地图（可缩放、点击查看聚类详情、筛选某方向论文）

### 场景 9：新研究方向发现（发散性分析）
**输入**：入库完成后的论文集合
**Agent 行为**：HDBSCAN 密度聚类 + 质心距离 + 论文相似度图 → 综合判断不属于任何聚类的论文 → LLM 评估是否构成潜在新方向 → 标记为"值得关注的新分支"
**产出**：潜在新方向列表（含支撑论文和探索建议）
**iOS 展示**：标注新方向论文（"这些论文不属主流聚类，可能代表新兴研究方向"），用户可关注或忽略

### 场景 10：研究全景可视化
**输入**：用户查看项目概览
**Agent 行为**：t-SNE/UMAP 降维所有论文向量 → 生成 2D/3D 坐标 → 叠加主题标签、时间轴、引用关系
**产出**：交互式研究全景图（论文分布 + 主题聚类 + 时间演变 + 引用网络）
**iOS 展示**：可缩放的 2D 地图 + 时间滑块（"2019→2024 研究热点如何迁移"）+ 点击论文查看详情

### 场景 11：以文搜文（引用追溯）
**输入**：用户提供种子论文（DOI / PDF / 标题）
**Agent 行为**：创建 CitationChaseAgent → Semantic Scholar API 获取引用关系 + arXiv/PubMed 并行补充 → LLM 评估每篇论文的相关性 → 并行下载+转换+索引 → 逐层追溯（默认2层，用户可指定，LLM可提前终止）
**产出**：Markdown 追溯报告 + 交互式引用网络图（iOS 可缩放/点击）
**iOS 展示**：引用网络图 + 追溯摘要（"从 X 出发，追溯 2 层，找到 35 篇相关论文，其中 18 篇高相关已入库"）

### 场景 12：历史消息处理
**输入**：Agent 重启后，Redis 中缓存的未处理历史消息
**Agent 行为**：创建 HistoryAgent → 分析消息集合 → 识别重复/过期/已处理/待处理 → 合并重复 → 归档过期 → 生成待办列表 → 通知主 Agent
**产出**：消息处理报告 + 合并摘要 + 待处理 action list
**iOS 展示**：Agent 重启后自动显示"您离开期间有 12 条新消息，其中 3 条已合并，5 条已处理，4 条待处理"

### 场景 14：中英学术术语翻译与术语库
**输入**：中文查询或新论文入库
**Agent 行为**：TranslationAgent 维护学术术语库 → 中文查询时自动翻译为准确学术英文 → 入库后自动从论文中提取新术语
**产出**：术语库（中英对照 + 学术语境 + 来源论文）
**iOS 展示**：术语库浏览页面 + 术语修正入口（用户可手动纠正翻译）

### 场景 13：PDF 获取追踪
**输入**：入库/追溯过程中部分论文 PDF 不可获取
**Agent 行为**：依次尝试 Semantic Scholar OA → arXiv direct → ScienceDirect → IEEE → publisher page → 全部失败后记录到 unavailable_pdfs 表
**产出**：PDF 不可获取列表（含尝试来源和失败原因）
**iOS 展示**：项目详情页展示"3 篇论文 PDF 暂不可获取"，可查看详情和建议（机构访问/联系作者/预印本替代）

### 4.1 完全主动的智能助理

- **请求驱动**：用户提问 → Agent 响应
- **订阅驱动**：每天自动检查订阅方向的新论文
- **主动发现**：分析用户画像，主动推荐热点方向和相关论文

### 4.1 完全主动的智能助理

- **请求驱动**：用户提问 → Agent 响应
- **订阅驱动**：每天自动检查订阅方向的新论文
- **主动发现**：分析用户画像，主动推荐热点方向和相关论文
- **记忆管理**：自动清理无用信息，保存关键知识，按需请求用户确认

### 4.2 终身记忆

| 记忆层级 | 生命周期 | 内容 |
|----------|----------|------|
| 短期记忆 | 当前对话窗口 | 最近 N 轮完整对话 |
| 中期记忆 | 当前任务 | 任务进度、中间结果、检查点 |
| 长期记忆 | 永久 | 研究偏好、重要发现、知识库 |
| 元记忆 | 永久 | 策略有效性、错误模式、用户偏好 |

### 4.3 可控透明的执行

- **Plan-then-Execute**：先生成方案 → 用户确认 → 逐步执行
- **子Agent 后台执行**：用户确认后，子Agent 接管入库全流程，主 Agent 继续对话
- **实时进度**：每步执行状态实时推送 iOS；每篇论文处理状态独立可查（下载中/已下载/转换中/已索引/失败及原因）
- **可暂停/恢复**：随时暂停任务，改参数后继续
- **可回溯**：完整操作日志（结构化 JSONL），LLM 决策全程可审计

### 4.4 流式全双工交互

- LLM 思考过程逐 token 实时展示
- 工具调用过程透明可见
- iOS 断开后 Agent 继续执行，重连时回放事件

### 4.5 智能知识组织（Embedding 驱动）

依托 doubao-embedding-vision (火山引擎) + ChromaDB 向量能力，在入库后自动对论文进行智能组织：

| 能力 | 技术 | 产品价值 |
|------|------|----------|
| **智能去重** | 向量相似度 >0.95 + DOI/标题硬约束 | 跨来源合并重复论文（预印本 vs 正式出版），保留权威版本 |
| **主题聚类** | K-means on embeddings + LLM 标签 | 自动发现研究方向，用户不必手动分类 |
| **新方向发现** | HDBSCAN 密度 + 质心距离 + 图分析 + LLM 综合判断 | 发现不属于主流聚类但可能代表新兴方向的论文 |
| **研究可视化** | t-SNE/UMAP 降维 + 时间轴 | 直观理解研究全景，追踪热点演变 |

这四项能力在入库完成后**自动触发**，结果直接呈现在 iOS 项目详情页。用户不需要理解"向量"是什么——看到的是"你的 50 篇论文分为 3 个研究方向"。

## 6. 交互模型

### 6.1 对话模式

- **iOS 客户端**：纯 WebSocket 全双工通道
- **消息类型**：7 大类 × 22 种子类（heartbeat/phase/thinking/message/tool/review/error），详见 WebSocket 协议文档
- **实时反馈**：LLM 逐 token 输出 + 工具调用进度 + 阶段状态

### 6.2 离线行为

- 用户断开后 Agent 继续执行未完成任务
- 事件缓存在 Redis，重连后回放
- 主动推送通过 APNs 通知 iOS

### 6.3 多项目支持

- 论文研究项目（paper_research）：arXiv, Semantic Scholar, PubMed 等
- 技术文档项目（tech_research）：GitHub, 官方文档, Stack Overflow 等（Phase 2）
- 跨项目全局搜索
