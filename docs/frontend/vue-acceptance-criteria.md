## 1. 验收说明

### 优先级定义

| 级别 | 含义 | 验收决策 |
|------|------|:---:|
| **P0** | 阻塞上线，核心功能 | 必须全部通过 |
| **P1** | 重要功能，影响体验 | 通过率 ≥ 95% |
| **P2** | 锦上添花，可延后 | 通过率 ≥ 80% |

### 验证方式

| 标记 | 方式 |
|------|------|
| 🖐 | 手动测试（按验收步骤操作） |
| 🤖 | 自动化测试（Vitest + Playwright） |
| 👁 | 视觉审查（对比设计稿） |
| 📐 | 工具检测（Lighthouse / axe / 浏览器 DevTools） |

## Phase 1：基础设施

**前提**：Vite + Vue 3 + TypeScript 项目已初始化，依赖已安装

| 编号 | 验收项 | 级别 | 方式 | 验收步骤 |
|------|--------|:---:|------|----------|
| **A-01** | 项目成功 build（无 TS/ESLint 错误） | P0 | 🤖 | `npm run build` 返回 0，无 error |
| **A-02** | `npm run dev` 启动开发服务器，HMR 正常 | P0 | 🖐 | 启动 → 修改 `.vue` 文件 → 浏览器无刷新即更新 |
| **A-03** | 路由守卫：未登录访问任意页面 → 重定向 `/login` | P0 | 🖐 | 清除 localStorage → 访问 `/` → 跳转登录页 |
| **A-04** | 登录流程完整：输入用户名+密码 → 调 `POST /api/auth/login` → 存 token 到 localStorage | P0 | 🖐 | 正确凭据登录 → localStorage 有 auth_token、refresh_token、user |
| **A-05** | 登录失败：错误密码 → Toast 提示"用户名或密码错误" | P0 | 🖐 | 输入错误密码 → Toast 提示 |
| **A-06** | Token 自动刷新：REST 请求返回 401 → 自动调 refresh → 重放原请求 | P0 | 🖐 | 等待 token 过期（或 mock）→ 请求自动重试成功 |
| **A-07** | Token 双过期：刷新也失败 → 跳转 `/login`，清 localStorage | P0 | 🖐 | mock 两次 401 → 跳转登录页 |
| **A-08** | 退出登录：清空 localStorage + 跳转 `/login` + 断开 WS | P0 | 🖐 | 点击退出 → localStorage 为空 → 跳转登录 |
| **A-09** | REST API 带 Bearer Token | P0 | 📐 | 打开 DevTools Network → 请求头含 `Authorization: Bearer xxx` |
| **A-10** | WS 连接携带 token | P0 | 📐 | 打开 DevTools WS → 连接 URL 含 `?token=xxx` |
