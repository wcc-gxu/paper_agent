# 智驭·研（AI Research Copilot）产品升级优化方案

## 1. 产品定位

智驭·研是一套面向计算机科研人员的 AI Research
Copilot，通过个人科研知识库、智能文献管理 Agent
和科研工作流编排能力，解决科研过程中知识难沉淀、文献调研效率低、长期研究上下文丢失以及科研写作规范化困难等问题。

产品核心不是替代 ChatGPT
的单次问答能力，而是围绕科研知识生命周期提供长期智能辅助：

-   知识获取：自动发现和整理相关论文、网页、音视频资料；
-   知识管理：构建个人科研知识库；
-   知识利用：基于可信知识进行检索、分析和写作辅助；
-   知识积累：长期学习用户研究方向、术语体系和写作习惯。

------------------------------------------------------------------------

# 2. 核心科研痛点

## 2.1 大模型文献幻觉

普通 LLM 可以总结论文，但无法保证引用真实性。

科研场景要求：

-   论文真实存在；
-   方法描述准确；
-   引用来源明确。

解决方案：

-   RAG grounding；
-   Citation Verification；
-   Crossref/arXiv 等外部校验；
-   低置信度拒答机制。

------------------------------------------------------------------------

## 2.2 中文科研需求与英文论文检索鸿沟

科研文献主要为英文，但用户经常使用中文描述研究问题。

例如：

用户输入：

"自动驾驶视觉对抗攻击"

数据库：

"adversarial attacks against autonomous driving perception systems"

解决方案：

-   使用支持跨语言语义空间的 embedding 模型；
-   中文 query 与英文论文保持语义接近；
-   Query Expansion 自动扩展专业关键词。

支持：

中文 → 英文术语 → 向量检索 → 相关论文。

------------------------------------------------------------------------

## 2.3 RAG知识库长期维护困难

简单 RAG 只能完成一次上传。

长期科研过程中：

-   新论文不断增加；
-   arxiv 和正式会议版本重复；
-   无关论文混入；
-   metadata缺失。

设计 Knowledge Agent：

负责：

-   文献搜索；
-   PDF下载；
-   内容解析；
-   智能切片；
-   去重；
-   embedding更新；
-   元数据维护。

------------------------------------------------------------------------

## 2.4 长流程科研任务执行困难

复杂科研任务通常包含多个步骤：

例如：

"调研近三年自动驾驶攻击方法"

需要：

1.  理解需求；
2.  搜索论文；
3.  筛选相关工作；
4.  分类总结；
5.  更新知识库；
6.  生成报告。

单次 LLM 无法可靠完成。

因此引入 Agent Workflow。

------------------------------------------------------------------------

# 3. 产品架构

## 3.1 Main Agent（主Agent）

作为用户入口。

负责：

-   自然语言意图识别；
-   判断任务复杂度；
-   制定执行计划；
-   调度不同专业 Agent。

简单问题直接调用 LLM。

复杂任务进入 Agent Workflow。

------------------------------------------------------------------------

## 3.2 Literature Agent

负责科研资料获取。

功能：

-   arXiv/Crossref检索；
-   论文下载；
-   论文元数据提取；
-   相关性筛选。

------------------------------------------------------------------------

## 3.3 Knowledge Agent

负责科研知识库生命周期管理。

功能：

-   PDF解析；
-   结构化切片；
-   embedding生成；
-   向量检索；
-   增量更新；
-   去重管理。

存储：

-   PostgreSQL：论文元数据、用户数据；
-   pgvector：向量检索。

------------------------------------------------------------------------

## 3.4 Research Agent

负责科研分析。

功能：

-   文献聚类；
-   方法对比；
-   研究趋势分析；
-   技术路线总结；
-   研究空白发现。

------------------------------------------------------------------------

## 3.5 Capture Agent

负责碎片化知识沉淀。

支持：

-   网页收藏；
-   实验记录；
-   会议笔记；
-   视频音频内容。

流程：

音视频 → ASR → 摘要 → 关键词 → 知识库。

------------------------------------------------------------------------

## 3.6 Writing Agent

辅助科研写作。

重点不是自动生成论文，而是提高规范性。

功能：

-   related work辅助；
-   citation检查；
-   学术表达优化；
-   模板化写作。

------------------------------------------------------------------------

# 4. 科研写作智能化设计

## 4.1 高质量论文模板学习

导师通常要求：

"参考某篇高水平论文的写法"。

系统支持：

上传优秀论文作为模板。

自动分析：

-   Introduction结构；
-   Contribution表达方式；
-   Method组织方式；
-   Experiment描述套路；
-   Conclusion总结方式。

形成：

Research Writing Template。

------------------------------------------------------------------------

## 4.2 术语词表管理

科研写作中术语一致性非常重要。

建立个人术语库：

例如：

occupancy prediction

统一：

占用预测

避免：

空置预测等错误翻译。

词表用于：

-   RAG查询扩展；
-   翻译校正；
-   写作检查。

------------------------------------------------------------------------

# 5. Agent Workflow

典型流程：

用户：

"调研自动驾驶视觉攻击最新进展"

↓

Main Agent

↓

Literature Agent

↓

Knowledge Agent

↓

Research Agent

↓

Writing Agent

↓

输出调研报告

这体现多Agent协作：

不同Agent负责不同专业能力，通过状态共享完成复杂任务。

------------------------------------------------------------------------

# 6. 工程优化方向

## 6.1 响应速度优化

问题：

所有请求进入Agent导致简单问题响应慢。

方案：

Intent Router：

-   简单问答 → LLM直接响应；
-   知识查询 → RAG；
-   复杂任务 → Agent Workflow。

------------------------------------------------------------------------

## 6.2 异步任务

长任务：

-   批量论文下载；
-   PDF解析；
-   embedding生成。

采用：

FastAPI + WebSocket + Celery + Redis。

实现：

后台执行和实时状态反馈。

------------------------------------------------------------------------

## 6.3 安全机制

包括：

-   Tool权限控制；
-   Prompt Injection防护；
-   外部文件隔离解析；
-   敏感操作确认；
-   Citation验证。

------------------------------------------------------------------------

问题：

不同科研任务需要不同能力。

形成：

Main Agent + 专业Agent架构。

------------------------------------------------------------------------


# 8. 关键技术亮点总结

1.  面向科研场景的长期知识管理 RAG；
2.  中文需求到英文论文的跨语言语义检索；
3.  自动化科研知识库维护；
4.  基于LangGraph的可控Agent Workflow；
5.  多Agent协作完成复杂科研任务；
6.  科研写作模板和术语一致性管理；
7.  多模态碎片知识沉淀能力。
